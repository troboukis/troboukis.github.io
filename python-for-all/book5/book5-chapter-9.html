Part One

Two-Level Architecture — List Page and Detail Page

The Hellenic Parliament publishes a searchable archive of daily legislative agendas. All 74 pages of that archive live under one URL pattern — but unlike a news feed, each item is a link to a separate detail page. You must scrape two layers to get the data.

Layer 1 — the list page. Each list page shows a batch of daily plans as links. The URL changes only in pageNo:

https://www.hellenicparliament.gr/Nomothetiko-Ergo/dailyplan?more=true&pageNo=1

https://www.hellenicparliament.gr/Nomothetiko-Ergo/dailyplan?more=true&pageNo=2

…

https://www.hellenicparliament.gr/Nomothetiko-Ergo/dailyplan?more=true&pageNo=74

Layer 2 — the detail page. Each list item carries a unique ID in its href. You extract that ID and build a second URL:

https://www.hellenicparliament.gr/Nomothetiko-Ergo/dailyplan?lpr_id=04734c57-7b71-4e5e-86e9-b43600adcc72

The detail page contains the full text of the daily plan, including the session date. Run the cell below to see how all 74 list-page URLs are built with a single range().

Part Three

      
Extracting the ID with Regex


      
The href from each list item looks like this:


      
/Nomothetiko-Ergo/dailyplan?lpr_id=04734c57-7b71-4e5e-86e9-b43600adcc72


      
You need only the value after lpr_id=. The pattern lpr_id=([\w-]+) captures it: [\w-] matches letters, digits, underscores, and hyphens; + means one or more. This is tighter than .*, which would consume everything including any parameters that might follow the ID.

        match.group(1) returns the first captured group — the text inside the first pair of parentheses in your pattern. match.group(0) would return the entire matched string, including lpr_id=.
      

      

        Why the full header set matters. The sec-fetch-dest, sec-fetch-mode, sec-fetch-site, and sec-fetch-user headers tell the server this is a real browser navigation, not a script. Government sites often check for these and return an empty page or a redirect when they are missing. Always copy the complete headers from your browser's Network tab — stripping them down is the most common reason a scraper silently breaks.
      

    


    

      
Part Four

      
A Bonus Skill — Parsing Greek Dates from Document Text


      
In this project the date is already available on the list page as 24/04/2026 — no extra work needed. But many Greek government documents store dates in the running text of the document body, in a format like:


      
Αθήνα, 22 Απριλίου 2025


      
You will encounter this pattern in court filings, ministerial decisions, and other official PDFs converted to text. The technique below extracts it: a regex matches "Αθήνα," followed by a day number, a Greek month name, and a four-digit year, and a dictionary maps the month name to its number.


      
One cleaning step is always required first: replace non-breaking spaces (\xa0) with regular spaces. Greek government documents often contain \xa0 characters from old word processors, and without this step the regex fails silently on many pages.

        Why \s+ instead of a single space? The regex uses \s+ (one or more whitespace characters) between each part of the date. Government documents are inconsistently spaced — sometimes a single space, sometimes two or three. \s+ handles all of them.

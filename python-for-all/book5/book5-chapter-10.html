Book Five · Chapter Ten

Playwright — Scraping Pages That Need a Real Browser

requests fetches raw HTML — it never runs JavaScript. Many modern websites build their content with JavaScript after the page loads, so requests sees an empty shell. Playwright solves this by launching a real browser that runs the JavaScript for you, then hands you the fully rendered page.

Part One

The Problem — What requests Cannot See

When you call requests.get(url), the server sends back the raw HTML file — exactly what was written on the server, before any JavaScript runs. On simple sites like news feeds, that raw HTML already contains all the content. But many modern sites work differently: they send a nearly empty HTML file and then use JavaScript, running in your browser, to fetch the data and build the page on the spot.

requests never runs that JavaScript. It gets the empty shell and stops there. Run the cell below to see what that looks like in practice.

Python · Try it

      


      
This is not a bug in your code. It is simply how those websites work. The fix is to use a tool that actually opens a browser, lets the JavaScript run, and only then reads the HTML.


      

        How to tell if a page needs a browser: open the page in Chrome, then right-click → View Page Source. If the source looks almost empty while the page looks full, the content is being added by JavaScript and requests will not see it.
      

    


    

      
Part Two

      
How Playwright Works


      
Playwright is a Python library that controls a real browser — Chromium, Firefox, or WebKit — from your code. You write Python, Playwright opens the browser, navigates to the URL, waits for the JavaScript to finish, and then gives you the full rendered HTML. From that point you use BeautifulSoup exactly as before.


      
The key difference from requests:


      
    
        
  • requests: sends one HTTP request, gets the raw HTML back, done. Fast, but blind to JavaScript.
    • 
        
  • Playwright: launches a full browser, loads the page, runs all the JavaScript, waits for everything to settle. Slower, but sees exactly what a human user would see.
    • 
      


      
Playwright also handles cookies, sessions, and login flows automatically — because it is a real browser. You never need to set sec-fetch-* headers or worry about bot detection: the browser sends all of that for you.


      

        When to use Playwright instead of requests: use requests first — it is much faster. Switch to Playwright only when the page content is missing or empty after a successful request.

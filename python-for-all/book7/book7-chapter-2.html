Why Visualise Data?

Numbers in a table tell you something. A chart tells you something immediately. In this book you will learn to turn raw data into charts and maps using Python's matplotlib library and, later, interactive geographic maps. This first chapter asks the fundamental question — why bother? — and then gets you drawing your first figure.

Part One

The Problem with Tables

Here are the monthly average temperatures in Athens, in degrees Celsius, for a recent year:

Month Jan Feb Mar Apr May Jun Jul Aug Sep Oct Nov Dec
°C 10 11 14 18 23 28 32 32 27 21 16 11

You can read the numbers, but you have to hold them all in your head to understand the shape of the year — the rise toward summer, the plateau in July and August, the descent back to winter. Now look at the same data as a line chart.

Part Two

      
Anatomy of a matplotlib Figure


      
Before you start drawing, it helps to understand the two main objects matplotlib uses.


      
A Figure is the whole canvas — the outermost container. A Axes (confusingly plural even for one chart) is the actual plot area inside the figure, where data is drawn. One figure can hold multiple axes side by side. For a single chart, you almost always create both at once with plt.subplots().

You will notice two styles in matplotlib code. The object-oriented style calls methods on ax directly: ax.plot(), ax.set_title(). The pyplot style calls functions on plt: plt.plot(), plt.title(). Both produce the same result. This course uses a mix — pyplot for quick one-off charts, the object-oriented style when you have multiple subplots.

Part Three

      
Labels, Titles, and the Grid


      
A chart without labels is almost always incomplete. At a minimum, every chart should have a title and labelled axes. The grid is optional but often helps readers estimate values.

A few things to notice:

      
    
        
  • marker="o" draws a dot at each data point.
    • 
        
  • color="steelblue" sets the line colour. Matplotlib accepts any CSS colour name, hex codes like "#2a6496", or shorthand letters like "r" for red.
    • 
        
  • alpha=0.5 makes the grid lines semi-transparent.
    • 
        
  • plt.tight_layout() adjusts spacing so nothing is clipped. Make it a habit to call it before plt.show().
    • 
      


      

        Marker options — pass any of these as marker=:


        "o" circle  ·  "s" square  ·  "^" triangle up  ·  "v" triangle down  ·  "D" diamond  ·  "*" star  ·  "+" plus  ·  "x" cross  ·  "." small dot  ·  None no marker (line only)
        


        Line style options — pass any of these as linestyle= (or the shorthand ls=):


        "-" solid (default)  ·  "--" dashed  ·  "-." dash-dot  ·  ":" dotted  ·  "None" no line (markers only)
        


        Colour options — pass any of these as color= (or the shorthand c=):


        Named: "steelblue", "tomato", "mediumseagreen", "goldenrod", "coral", "royalblue", "slategray", "darkorange" — any of the 148 CSS colour names work.

        Hex code: "#2a6496", "#e74c3c", etc.

        Shorthand letters: "r" red, "g" green, "b" blue, "c" cyan, "m" magenta, "y" yellow, "k" black, "w" white.

Two Lines on One Chart


      
You can call ax.plot() more than once on the same axes. Each call adds a new line. When you have multiple lines, add a legend so the reader knows which is which.

Saving a Chart and Publishing It on a Web Page


      
Running a chart in a notebook is one thing. Putting it on an actual web page is another. matplotlib makes it straightforward: instead of calling plt.show(), call plt.savefig() with a file path.

For web publishing, SVG is often a better choice than PNG. An SVG file is a vector format — it scales to any size without becoming blurry, and it is usually smaller than a high-DPI PNG. Change the extension to .svg and matplotlib handles the rest. The HTML embed is identical.


      

        PNG vs SVG: use PNG when the chart contains many data points (scatter plots with thousands of dots render faster as PNG). Use SVG for bar charts, line charts, and anything you might need to display at different sizes — it will look sharper on retina screens and when printed. Both embed in HTML with a plain <img> tag.

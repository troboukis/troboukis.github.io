Geographic Data and Maps

Every location on Earth can be described with two numbers. Every boundary — a country, a municipality, a flood zone — can be described as a list of coordinate pairs. Once you understand that, geographic data stops being mysterious and becomes just another dataset. In this chapter you will learn how location data is structured, how to work with it in Python, and how to build interactive maps.

Part One

The Language of Place — Latitude and Longitude

Every point on Earth has two coordinates: latitude (how far north or south of the equator, from −90° to 90°) and longitude (how far east or west of the prime meridian, from −180° to 180°). Together they pinpoint any location uniquely.

Athens37.983823.7275
Thessaloniki40.640122.9444
Heraklion35.338725.1442

In Python, these are just numbers. A city is a dictionary. A list of cities is a list of dictionaries — the same structure you have been working with since Book 4.

Part Two

      
GeoJSON — Geography as Data


      
GeoJSON is the standard format for geographic data on the web. It is JSON — the same format you learned in Book 6 — with a specific structure for describing points, lines, and polygons.


      
A Point is a single location. A LineString is a sequence of points forming a path (a road, a river). A Polygon is a closed area (a country, a park). All three wrap their geometry in a Feature, and a list of Features is a FeatureCollection.

A GeoJSON file is just a text file with a .geojson extension. You read it with json.load(), and you work with it as a regular Python dictionary. Boundary files for every country and administrative division in the world are available for free from sources like Natural Earth and the EU's GISCO service.


      

        Finding GeoJSON files: Natural Earth (naturalearthdata.com) provides country and region boundaries at multiple scales. The EU's GISCO service provides administrative boundaries for Greece and all EU member states. Search for "GeoJSON Greece municipalities" to find pre-made files ready to use.

Part Five

      
Choropleth Maps — Colouring Regions by Value


      
A choropleth map colours geographic regions by a numeric value. You have seen them in news coverage of election results, unemployment rates, or COVID case rates. The two inputs are a GeoJSON boundary file and a dataset that shares an identifier with the boundaries.

The key line is key_on="feature.properties.NUTS_ID". This tells folium which field inside the GeoJSON properties to use as the join key — it must match a column in your DataFrame. Every boundary file uses different property names, so always inspect the GeoJSON first to find the right field.


      

        Colour scales: folium supports all ColorBrewer scales. "YlOrRd" (yellow-orange-red) works well for rates where higher values are bad. "RdYlGn" (red-yellow-green) is good when higher is better. Avoid rainbow scales ("Spectral") for continuous data — they are hard to interpret and inaccessible to colour-blind readers.

Put both files — your article HTML and greece_map.html — in the same folder and the embed will work. If you move the map file to a subfolder, update the src path accordingly (e.g. src="maps/greece_map.html").


      

        iframe vs inline: an iframe is a separate browsing context — the map's Leaflet.js, CSS, and event handlers are isolated from your page's code. This is the right approach for Folium maps because Folium generates its own HTML structure. Trying to paste the map's raw HTML directly into your article page will almost always break either the map or your page's layout.

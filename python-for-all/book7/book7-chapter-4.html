Scatter Plots and Histograms

Not all stories are about how things change over time. Sometimes the story is about relationships — does higher education spending correlate with better outcomes? Sometimes it is about shape — are prices clustered tightly or spread widely? Scatter plots and histograms answer those questions.

Part One

Scatter Plots — Revealing Relationships

A scatter plot places each observation as a dot on a two-dimensional plane. The x position encodes one variable, the y position encodes another. If a pattern emerges — a diagonal cloud, a curve, a cluster — there is a relationship worth investigating.

        Correlation is not causation: a scatter plot shows that two things move together. It does not tell you which one causes the other, or whether both are caused by a third factor. Scatter plots are for spotting patterns, not for proving cause and effect.
      

    


    

      
Part Two

      
Using Colour and Size as Extra Dimensions


      
A basic scatter plot encodes two variables. But a dot can carry more information: its colour can encode a category, and its size can encode a third numeric variable. This is how classic "bubble charts" work.

This chart encodes four variables: x (income), y (unemployment), colour (income again — reinforcing the pattern), and size (population). The cmap="RdYlGn" parameter uses a red-yellow-green colour map — low values in red, high in green. plt.colorbar() adds the legend strip that explains the colour scale.


      

        Don't overdo it: encoding four or five variables on one chart is only useful if each dimension adds a distinct insight. If colour and x axis tell the same story, the colour is redundant. Here it deliberately reinforces the message — higher income, greener dot, lower unemployment.
      


      

        Colour map options — pass any of these as cmap=:


        Sequential (low → high): "Blues" "Greens" "Reds" "Oranges" "viridis" "plasma" "YlOrRd" "YlGnBu"

        Diverging (low ↔ high around a midpoint): "RdYlGn" "RdBu" "coolwarm" "bwr"

        Perceptually uniform (good default choice): "viridis" "cividis" "inferno"


        Append "_r" to any name to reverse the direction — e.g. "Blues_r" goes from dark to light.
      

    


    

      
Part Three

      
Histograms — Showing Distributions


      
A histogram answers a different question from a scatter plot. Instead of "how do two variables relate?", it asks "how are values distributed?" You give it a single list of numbers and it divides them into bins, counting how many fall into each.

The shape reveals something a mean or median cannot: there are two clusters of rents — a main market around €800 and a smaller premium market around €1,400. Reporting only the average would obscure this bimodal structure.


      
The bins= parameter controls how many bars the range is divided into. Too few bins and you lose detail; too many and noise dominates. Try changing it to bins=5 and bins=60 to see the effect.


      

        Mean vs. median in skewed distributions: when a histogram has a long tail to the right (a few very expensive apartments), the mean is pulled upward by those outliers. The median — the value that splits the distribution in half — is more representative. Both numbers together tell a richer story.
      

    


    

      
Part Four

      
Subplots — Multiple Charts in One Figure


      
Often the most informative layout is to place two or three related charts side by side. plt.subplots(nrows, ncols) creates a grid of axes. The second return value is an array of axes objects — one per cell in the grid.

        What you learned in this chapter: scatter plots reveal relationships between two numeric variables; colour and size can encode two more. Histograms reveal the shape of a distribution — something a mean can never show. Subplots let you place multiple charts in one figure for direct comparison; share axes with sharey() or sharex() to keep comparisons fair.
